1. 背景 · 目标 · 非目标

现状(M10 已交付):平台已具备完整的线上运行监控——运行量/延迟/TTFT/Token/阶段耗时、失败/兜底/超时/降级、修复率,以及 Trace/Session 全链路下钻。这些回答「系统跑得稳不稳」。

缺口:没有任何指标回答「回答好不好」。现有信号的边界必须认清:

现有信号它不是为什么
低分召回(rag.quality.low_recall)≠ Context Recall只是检索分数阈值启发式,不知道「该召回什么」
引用数量/覆盖≠ Faithfulness带了引用不代表内容忠于引用
启发式可信度(引用分最大值)≠ 事实正确率是检索分数的代理,不判断内容对错
失败率低≠ 回答质量高「正常返回了」和「返回的是好答案」是两回事

最危险的隐性问题:一个应用可以 兜底率 0%、失败率 0%(看板全绿),但忠实度 60%——在大量一本正经地胡说八道。现有看板抓不到,这正是本设计要解决的第一价值。

目标

① 在真实流量上持续评估回答质量(免标注,常开)
② 建立 gold 评测集,可重复严格评测(正确率/召回率)
③ 任何改动(模型/Prompt/参数/知识库)可 A/B 对比、防回退,并接入上线门禁
④ 坏样本自动沉淀为知识缺口,分诊后补库/进评测集,形成数据飞轮
⑤ 与 Trace 深度联动:每个质量分可下钻到链路,每条 trace 可一键重放/入池/入集

非目标(本期不做)

· 人工众包标注平台(标注在本系统内轻量完成)
· 真实成本 USD 评估(依赖定价体系,延续 016 决策继续延后)
· 线上真实流量灰度 A/B(本期 A/B 只在评测集离线跑)
· 自动无人审入库(红线:LLM 草拟必须人审)
· 多租户/权限细分(沿用现状单 admin)

3. 核心概念与三大指标计算

术语定义
在线评测(reference-free)对真实流量抽样,LLM 裁判打分,不需要标准答案。指标:忠实度 / 回答相关性 / 上下文精确率
gold / 标准答案评测集用例里人工审定的正确答案(gold answer)与应命中文档(gold docs)。是 reference-based 指标的前提,尺子不准量什么都错,需人工审核+随文档更新维护
离线评测(reference-based)在 gold 题库上跑,可算 Context Recall / Answer Correctness / Citation Correctness / NDCG / MRR
根因分诊答得差的三种根因:缺内容(该补库)/ 检索问题(库里有但没召回,该调参)/ 生成问题(召回对了答砸了,该改 Prompt/模型)。只有「缺内容」才补库
重放 replay把一条 trace 的原始问题重新走一遍编排(原配置=重现 / 候选版本=单条 A/B),产出 preview trace 并排对比

3.1 忠实度 Faithfulness — 回答有没有「编」

输入:回答 + 本次检索片段正文。算法(Ragas 式):

1LLM 把回答拆成独立事实主张 → 2逐条问「能否从片段推出?」(是/否) → 3忠实度 = 被支持主张数 / 总主张数

例:回答含 3 条主张,片段只支持 2 条 → 67 分。抓不到:片段本身过时/错误时,忠实的错误答案仍高分(那是 Correctness 的事,归离线评测)。

3.2 回答相关性 Answer Relevancy — 有没有「答到点上」

输入:问题 + 回答 + embedding 模型。算法:

1只给 LLM 看回答,反推「它在回答什么问题」×N 个 → 2反推问题与真实问题做 embedding 余弦相似 → 3取平均

切题→反推问题≈原问题→高分;跑题/敷衍(「这个我不清楚」)→反推不出→低分。抓不到:切题但事实错误。

3.3 上下文精确率 Context Precision — 召回的「相不相关、排得好不好」

输入:问题 + 召回的每段正文(reference-free 变体)。算法:

1逐段问 LLM「这段对回答该问题有用吗?」 → 2按排名加权:在每个相关片段出现的位置 k 算 precision@k,对所有相关位置取平均

例:5 段判定 [✓,✗,✓,✗,✗] → (1/1 + 2/3)/2 ≈ 0.83;同样 2 段相关若排在 4、5 位,分数显著更低(惩罚 rerank 没排好)。抓不到:漏召回(Recall,需 gold docs,归离线评测)。

3.4 三指标组合 → 定位坏在哪一环

精确率忠实度相关性诊断去修
低低尚可检索捞回垃圾,模型被带偏检索/rerank/清库
高低高资料对,模型仍编reply Prompt / 换模型
高高低没答到点,问题被改歪rewrite / intent
低低低系统性差该应用配置/知识库整体

3.5 离线评测追加指标(需 gold)
